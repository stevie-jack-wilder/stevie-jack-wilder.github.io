Explain control flow and loops using an example process from everyday life.

Control flow allows us to dictate the order in which statements, instructions or functions execute their assign actions. Let's think about control flow in terms of our day-to-day life. I'm making dinner, it's going to be a simple stir-fry. We have a few steps to complete.

  • Dice vegetables and chicken.
  • Make sauce, prep noodles.
  • Add oil to pan.
  • Cook chicken.
  • Add veg, noodles, sauce to pan.
  • Stir pan, open beer, finish beer, serve.

Control flow is the part of my brain that knows I need to dice my vegetables and chicken before they go in a pan, and before anything goes in a pan, there needs to be oil in there. Oh, and we probably want the oil heated, too! Without control flow, I might just stir an empty cold pot, open a beer, finish the beer, and serve myself a lovely plate of nothing, then go and dice some veg.All of the correct instructions are in there, but without creating a control loop, it won't be executed in the desired order.

Loops work in a similar way, instead, we are thinking about repetitive tasks that need to be executed a number of times before we consider a job (or 'function') to be done. We can dictate how many loops of an action is executed before it finishes. For example, when cutting vegetables for my stirfry, I need to move my knife in a certain action X times to get Y pieces of delicious brocollini preppped for the pan. But I also need to know when I have enough and when to stop.


Describe what the DOM is and an example of how you might interact with it.

The DOM is the glue, as Kirupa describes, that holds together the html, css, and javascript. It is an object representation of your entire project and acts as a gateway to the browser. It is a collection of nodes: elements, attributes, text content, comments, docs, and other things. The mapping is done via Node > Element > HTMLElement. every HTMLElement has a JS equivalent. Such as an anchor in html (a)is HTMLAnchorElement in JS.


What is the difference between accessing data from arrays and objects?

An Object stores a collection of data, and we call this data a set of 'properties'. Each property has a key and a value. For example:

    var object = {
    key: 'value'
    };

In your phone's Contacts List, we could think of each person as an object with nesting unique properties, such as name, number, address, alternate contact. We could also think of these properties as characteristics of the object. For example:

    var person = {
    name: 'Stevie',
    age: 33,
    lovesCats: true
    };

Arrays also store data but as a list of multiple items in a single variable. Arrays are useful for creating ordered collections where each item is assigned a numerical value. It is important to remember that we start with zero when working with most coding languages. So, the first word in this paragraph, "Array", would be assigned the number[0]. Arrays can contain strings, numbers, bools, objects, or even other arrays!

So, what's the answer? In short, we access data from Objects when we want to find a characteristic, or property, within that Object. We access data from Arrays when we want to find the numberical position of an item within a variable. This number[0] allows us to interact with this item by identifying it in functions.
